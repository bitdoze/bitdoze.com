## Project Overview

The repo contains a working implementation with these parts:

- **Agent factory** — wires model, tools, memory, and session settings
- **Discord bot** — handles mentions and replies with typing indicators
- **Heartbeat** — runs every 30 minutes for periodic checks
- **Cron jobs** — hot-reloads from `workspace/CRON.yaml` every 10 minutes
- **Context injection** — loads personality and rules from markdown files

## Configuration

### 1. Environment Variables

Create `.env`:

```bash
cp .env.example .env
```

Edit it:

```
DISCORD_BOT_TOKEN=YOUR_DISCORD_TOKEN
OPENAI_API_KEY=YOUR_MODEL_API_KEY
GITHUB_ACCESS_TOKEN=OPTIONAL_GITHUB_TOKEN
```

### 2. Main Config (`config.yaml`)

```yaml
model:
  provider: openai_like
  id: stepfun/step-3.5-flash:free
  base_url: https://openrouter.ai/api/v1
  api_key_env: OPENAI_API_KEY

discord:
  token_env: DISCORD_BOT_TOKEN

memory:
  mode: automatic
  db_file: data/bitdoze.db
  add_history_to_context: true
  read_chat_history: true
  enable_session_summaries: true
  summary_prompt: |
    You are generating a session summary. Return ONLY a JSON object.
    Required keys:
      - summary: string
      - topics: array of strings (optional)
    Use lowercase keys exactly.

toolkits:
  web_search:
    enabled: true
    backend: duckduckgo
  hackernews:
    enabled: true
  website:
    enabled: true
  github:
    enabled: true
    token_env: GITHUB_ACCESS_TOKEN
  youtube:
    enabled: true
  file:
    enabled: true
    base_dir: workspace
  shell:
    enabled: true
    base_dir: workspace
  discord:
    enabled: true

heartbeat:
  enabled: true
  interval_minutes: 30
  quiet_ack: HEARTBEAT_OK
  prompt_path: workspace/HEARTBEAT.md

cron:
  path: workspace/CRON.yaml

context:
  add_datetime: true
  timezone_identifier: Europe/Bucharest
  agents_path: workspace/AGENTS.md
  user_path: workspace/USER.md
  memory_dir: workspace/memory
  long_memory_path: workspace/MEMORY.md

agents:
  default: main
  definitions:
    - name: main
      tools: [web_search, hackernews, website, github, youtube, file, shell, discord]
```

## Discord Bot Setup

1. Create an app at [discord.com/developers](https://discord.com/developers)
2. Add a bot, copy the token
3. Enable **Message Content Intent**
4. Invite to your server with required permissions

## Workspace Files (Personality + Memory)

These files shape how the agent behaves:

| File | Purpose |
|------|---------|
| `workspace/SOUL.md` | Identity and tone |
| `workspace/AGENTS.md` | Operating rules |
| `workspace/USER.md` | Your preferences |
| `workspace/MEMORY.md` | Long-term memory |
| `workspace/memory/YYYY-MM-DD.md` | Daily notes |

<Accordion label="Example SOUL.md" group="examples">
```markdown
# SOUL.md - Who You Are

**Be genuinely helpful, not performatively helpful.** Skip the "Great question!" filler. Just help.

**Have opinions.** You're allowed to disagree or find things boring. A personality-free assistant is just a search engine with extra steps.

**Be careful.** You have access to someone's life. Treat it with respect. Ask before acting externally.
```
</Accordion>

## Skills (Optional)

Skills keep instructions modular. Store them in `skills/` and load only when needed.

```
skills/
  web-research/
    SKILL.md
```

Example `SKILL.md`:

```markdown
---
name: web-research
description: Focused web research using search and website tools.
---

Use WebSearchTools and WebsiteTools to research topics. Summarize with sources.
```

```bash
uv run main.py --config config.yaml
```

In Discord, mention your bot:

```

```

## Heartbeat (Periodic Checks)

Heartbeat runs every 30 minutes and loads `workspace/HEARTBEAT.md`. If it returns `HEARTBEAT_OK`, it stays silent.

This mirrors the [Clawdbot](/clawdbot-setup-guide/) pattern for proactive monitoring.

## Cron Jobs (Scheduled Tasks)

Define jobs in `workspace/CRON.yaml`:

```yaml
enabled: true
timezone: Europe/Bucharest
channel_id: 123456789012345678
jobs:
  - name: daily-status
    cron: "0 9 * * *"
    agent: main
    message: "Send a daily status update."
    deliver: true
    session_scope: isolated
```

Jobs hot-reload every 10 minutes — no restart needed.

## Systemd Service (24/7)

Create `~/.config/systemd/user/bitdoze-bot.service`:

```ini
[Unit]
Description=Bitdoze Bot
After=network.target

[Service]
WorkingDirectory=/home/dragos/projects/bitdoze-bot
ExecStart=/home/dragos/.local/bin/uv run main.py --config config.yaml
Restart=always
RestartSec=5
Environment=PYTHONUNBUFFERED=1
EnvironmentFile=/home/dragos/projects/bitdoze-bot/.env

[Install]
WantedBy=default.target
```

Enable:

```bash
systemctl --user daemon-reload
systemctl --user enable --now bitdoze-bot
systemctl --user status bitdoze-bot
```

## Troubleshooting

<Accordion label="Bot not responding" group="troubleshooting">
- Check `DISCORD_BOT_TOKEN` in `.env`
- Ensure Message Content Intent is enabled
- Use `journalctl --user -u bitdoze-bot -f` for logs
</Accordion>

<Accordion label="Replies too long" group="troubleshooting">
Discord limits messages to 2000 characters. The bot auto-chunks longer responses.
</Accordion>

<Accordion label="Session summary errors" group="troubleshooting">
Make sure the summary prompt enforces JSON with a `summary` key.
</Accordion>

import argparse
from pathlib import Path
from dotenv import load_dotenv
from bitdoze_bot.discord_bot import run_bot

def main() -> None:
    parser = argparse.ArgumentParser(description="Bitdoze Bot (Agno + Discord)")
    parser.add_argument("--config", default="config.yaml")
    parser.add_argument("--env-file", default=".env")
    args = parser.parse_args()
    
    if Path(".env").exists():
        load_dotenv(".env")
    if Path(args.env_file).exists():
        load_dotenv(args.env_file)
    
    run_bot(args.config)

if __name__ == "__main__":
    main()

<Accordion label="bitdoze_bot/agents.py (Agent Factory)" group="code">
```python
from agno.agent import Agent
from agno.db.sqlite import SqliteDb
from agno.models.openai.like import OpenAILike
from agno.tools.github import GithubTools
from agno.tools.hackernews import HackerNewsTools
from agno.tools.discord import DiscordTools
from agno.tools.file import FileTools
from agno.tools.shell import ShellTools
from agno.tools.websearch import WebSearchTools
from agno.tools.website import WebsiteTools
from agno.tools.youtube import YouTubeTools

from bitdoze_bot.config import Config

TOOL_MAP = {
    "web_search": lambda: WebSearchTools(),
    "hackernews": lambda: HackerNewsTools(),
    "website": lambda: WebsiteTools(),
    "github": lambda c: GithubTools(
        access_token=os.getenv("GITHUB_ACCESS_TOKEN") or None
    ),
    "youtube": lambda: YouTubeTools(),
    "file": lambda c: FileTools(base_dir=Path("workspace")),
    "shell": lambda c: ShellTools(base_dir=Path("workspace")),
    "discord": lambda c: DiscordTools(
        bot_token=os.getenv("DISCORD_BOT_TOKEN")
    ),
}

def build_agents(config: Config) -> Agent:
    model = OpenAILike(
        id=config.get("model", "id", default="gpt-4o-mini"),
        api_key=os.getenv(config.get("model", "api_key_env", default="OPENAI_API_KEY")),
        base_url=config.get("model", "base_url", default=None),
    )
    
    tools = []
    for name in config.get("agents", "definitions", 0, "tools", default=[]):
        if name in TOOL_MAP:
            tools.append(TOOL_MAP[name](config))
    
    db_path = Path(config.get("memory", "db_file", default="data/bitdoze.db"))
    db_path.parent.mkdir(parents=True, exist_ok=True)
    
    return Agent(
        name="main",
        model=model,
        tools=tools,
        db=SqliteDb(db_file=str(db_path)),
        markdown=True,
        update_memory_on_run=True,
        add_history_to_context=True,
        read_chat_history=True,
        enable_session_summaries=True,
        add_session_summary_to_context=True,
        add_datetime_to_context=True,
    )
```
</Accordion>

<Accordion label="workspace/AGENTS.md" group="code">
```markdown
# AGENTS.md - Bitdoze Workspace

## Every Session (automatic)

1. Read `SOUL.md` — this is who you are
2. Read `USER.md` — this is who you're helping
3. Read `memory/YYYY-MM-DD.md` (today + yesterday)
4. In main sessions: also read `MEMORY.md`

## Memory Model

- **Daily logs:** `workspace/memory/YYYY-MM-DD.md` — raw notes
- **Long-term:** `workspace/MEMORY.md` — curated memory

Write key decisions to daily logs. Promote important items to `MEMORY.md`.

## Discord Rules

- Respond only when mentioned
- No markdown tables — use bullet lists
- Wrap URLs in `< >` to suppress previews

## Tools & Skills

- Skills live in `skills/` at repo root
- Use Agno's format: `SKILL.md` with YAML frontmatter
```
</Accordion>
