This guide shows how to build a **Discord-first AI bot** with **Agno** and run it 24/7.  
It started as a Clawdbot-style replacement and now supports **multi-agent teams** with **workspace-based agent folders**.

<Notice type="info" title="What You’ll Build">
A Discord bot that responds on mention, uses tools (web, GitHub, file, shell), stores memory, learns over time, runs scheduled tasks, and can switch between a solo coordinator and a full delivery team.
</Notice>

## Why This Setup

- You keep control (self-hosted, your tokens, your data path).
- You can start simple (`main` agent) and scale to specialized teams.
- You get practical automation: heartbeat checks + cron jobs.
- You get observability logs and test coverage for stability.

## Prerequisites

<ListCheck>
<ul>
<li>Python 3.12+</li>
<li>Discord bot token</li>
<li>Model API key (OpenAI-compatible provider)</li>

</ul>

## Quick Start

```bash
git clone https://github.com/bitdoze/bitdoze_bot.git
cd bitdoze_bot
cp config.example.yaml config.yaml
cp .env.example .env
uv sync
uv run main.py --config config.yaml
```

## Core Features

- **Agents + Teams**: Agno `Agent` and `Team` support
- **Workspace Agents**: folder-based agent loading from `workspace/agents/<name>/`
- **Memory + Learning**:
  - shared SQLite memory DB
  - per-agent LearningMachine stores (`user_profile`, `user_memory`, etc.)
- **Automation**:
  - heartbeat interval checks
  - cron jobs from `workspace/CRON.yaml`
- **Observability**:
  - run start/end logs
  - timing, model, token metrics (when available)
  - team delegation paths
- **Tests**:
  - registry wiring (agents/teams/aliases)
  - routing behavior
  - delegation helper checks

## Configure `.env`

```env
DISCORD_BOT_TOKEN=YOUR_DISCORD_TOKEN
GLM_API_KEY=YOUR_MODEL_API_KEY
GITHUB_ACCESS_TOKEN=OPTIONAL_GITHUB_TOKEN
```

## Configure `config.yaml`

```yaml
model:
  provider: openai_like
  id: glm-4.7
  base_url: https://api.z.ai/api/coding/paas/v4
  api_key_env: GLM_API_KEY
  structured_outputs: off

discord:
  token_env: DISCORD_BOT_TOKEN

memory:
  mode: automatic
  db_file: data/bitdoze.db
  add_history_to_context: true
  read_chat_history: true
  search_session_history: true
  add_memories_to_context: true
  enable_session_summaries: false

learning:
  enabled: true
  mode: always
  add_to_context: true
  stores:
    user_profile: true
    user_memory: true
    session_context: false
    entity_memory: false
    learned_knowledge: false
    decision_log: false

toolkits:
  web_search: { enabled: true, backend: duckduckgo }
  hackernews: { enabled: true }
  website: { enabled: true }
  github: { enabled: true, token_env: GITHUB_ACCESS_TOKEN }
  youtube: { enabled: true, languages: [en] }
  file: { enabled: true, base_dir: workspace }
  shell: { enabled: true, base_dir: workspace }
  discord: { enabled: true }

context:
  add_datetime: true
  timezone_identifier: Europe/Bucharest
  agents_path: workspace/AGENTS.md
  user_path: workspace/USER.md
  memory_dir: workspace/memory
  long_memory_path: workspace/MEMORY.md
  main_session_scope: dm_only

agents:
  workspace_dir: workspace/agents
  aliases:
    engineer: software-engineer
    software-enginner: software-engineer
  default: main
  definitions:
    - name: main
      tools: [web_search, hackernews, website, github, youtube, file, shell, discord]
    - name: research
      tools: [web_search, hackernews, website, github, youtube, file, shell]
  routing:
    rules:
      - agent: delivery-team
        contains: ["manage", "orchestrate", "team:"]
        starts_with: ["team:"]
      - agent: software-engineer
        contains: ["build"]
      - agent: architect
        contains: ["plan"]

teams:
  definitions:
    - name: delivery-team
      members: [architect, software-engineer]
      respond_directly: true
      determine_input_for_members: true
      delegate_to_all_members: false
      add_member_tools_to_context: true
      add_team_history_to_members: true
      num_team_history_runs: 5
```

<Notice type="warning" title="Important Team Flag">
Do not set `delegate_to_all_members: true` together with `respond_directly: true` unless you explicitly want broadcast delegation behavior. Agno will warn and adjust behavior.
</Notice>

## Add New Agents (No Code Changes)

Create a folder: `workspace/agents/<agent-name>/`

`workspace/agents/product-manager/agent.yaml`
```yaml
name: product-manager
enabled: true
model:
  id: glm-4.7
  base_url: https://api.z.ai/api/coding/paas/v4
  api_key_env: GLM_API_KEY
skills: []
```

`workspace/agents/product-manager/AGENTS.md`
```md
# Product Manager Agent

Focus on scope, priorities, acceptance criteria, and delivery risk.
```

Then include it in a team:

```yaml
teams:
  definitions:
    - name: delivery-team
      members: [architect, software-engineer, product-manager]
```

## Use It in Discord

Mention the bot normally:
```text
@YourBot help me design a migration plan
```

Force a specific target:
```text
@YourBot agent:delivery-team Build and validate this feature end-to-end
@YourBot agent:architect Propose architecture options
@YourBot agent:software-engineer Implement this with tests
```

Trigger team by rule:
```text
@YourBot team: build release checklist for this feature
```

## Heartbeat + Cron

`workspace/CRON.yaml`
```yaml
enabled: true
timezone: Europe/Bucharest
channel_id: 123456789012345678
jobs:
  - name: daily-status
    cron: "0 9 * * *"
    agent: main
    message: "Send a daily status update."
    deliver: true
    session_scope: isolated
```

Heartbeat uses `workspace/HEARTBEAT.md` every 30 minutes and suppresses empty updates with `HEARTBEAT_OK`.

## Production (systemd)

`~/.config/systemd/user/bitdoze-bot.service`
```ini
[Unit]
Description=Bitdoze Bot
After=network.target

[Service]
WorkingDirectory=/home/dragos/projects/bitdoze-bot
ExecStart=/home/dragos/.local/bin/uv run main.py --config config.yaml
Restart=always
RestartSec=5
Environment=PYTHONUNBUFFERED=1
EnvironmentFile=/home/dragos/projects/bitdoze-bot/.env

[Install]
WantedBy=default.target
```

```bash
systemctl --user daemon-reload
systemctl --user enable --now bitdoze-bot
journalctl --user -u bitdoze-bot -f
```
