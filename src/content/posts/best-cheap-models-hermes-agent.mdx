It runs 1.6 trillion total parameters with 49 billion activated per token and supports both thinking and non-thinking modes.

### Setting up OpenCode Go in Hermes

Add the OpenCode Go endpoint to `~/.hermes/.env`:

```bash
echo "OPENAI_BASE_URL=https://opencode.ai/zen/go/v1/chat/completions" >> ~/.hermes/.env
echo "OPENAI_API_KEY=your-opencode-go-key" >> ~/.hermes/.env
```

Then set the model:

```bash
hermes config set model opencode-go/minimax-m2.7
```

Switch models anytime:

```bash
hermes model
```

### Which one should you pick?

**On a tight budget:** MiniMax M2.7. At $0.30/M input, you can run Hermes all month for the price of a coffee. The performance is good enough for most daily tasks — reminders, quick questions, file management, simple research.

**Need long context:** DeepSeek V4 Pro. The 1M context window at $0.435/M input is hard to beat. If your Hermes conversations get long or you process big files, this is the one.

**Want the strongest agent:** MiMo V2.5 Pro or GLM 5.1. Both match Claude Opus 4.6 on agent benchmarks. MiMo V2.5 Pro is slightly better at sustained long-horizon tasks with its token efficiency. GLM 5.1 has the edge on pure coding with its 58.4% SWE-Bench Pro score.

**Do not want to choose:** OpenCode Go at $10/month gives you all five models. Switch between them based on the task.

## What I actually run

My Hermes setup uses MiniMax M2.7 as the default model for everyday chat and quick tasks. For complex coding jobs and research tasks, I switch to GLM 5.1 or MiMo V2.5 Pro. DeepSeek V4 Pro handles anything that needs a long context window.

The fallback configuration looks like this:

```bash
hermes config set model minimax/minimax-m2.7
```

When I need more power for a specific task:

```bash
hermes model
# Select GLM 5.1 or MiMo V2.5 Pro
```

For most Hermes users, starting with MiniMax M2.7 and switching up when needed keeps costs low without sacrificing capability.
